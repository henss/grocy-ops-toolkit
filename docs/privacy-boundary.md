# Privacy Boundary For grocy-ops-toolkit

Default stance: public examples must be synthetic, minimal, and source-neutral.

Allowed:

- generic Grocy APIs, schemas, and helper logic
- invented fixtures with placeholder names and values
- public Grocy documentation links
- reduced examples that preserve behavior without private context

Not allowed:

- real Grocy database exports, product catalogs, stock levels, or backup manifests
- credentials, tokens, internal URLs, local filesystem paths, and production config
- screenshots or logs copied from private systems
- private project names, account identifiers, or operational planning context

Any GitHub repo creation, visibility change, remote addition, push, or package publication is approval-gated outside this scaffold.
